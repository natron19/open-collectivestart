# CollectiveStart Demo

> Tell me about your working community. Get your founding starter pack.

CollectiveStart helps co-op founders generate a Founding Starter Pack — three AI-drafted artifacts to kick off the real conversations:

1. **Founding Conversation Summary** — values, governance preferences, and alignment questions your team still needs to answer
2. **Cooperative Business Model Canvas** — customer segments, revenue streams, costs, and ownership structure
3. **Legal Form Comparison** — worker cooperative vs. multi-member LLC vs. benefit corporation, tailored to your state

All output carries a prominent "Draft for lawyer review" banner. The goal is to give founders a concrete document to react to, not a finished plan.

---

> **Built on [Open Demo Starter](https://github.com/natron19/open-demo-starter)** — a minimal Rails 8 + Gemini boilerplate for single-purpose AI demos.

---

## Why I built this

Co-op founders often stall at the "blank page" stage. The legal and governance questions are real, but there is no obvious starting artifact. Most founders either dive into a lawyer's office cold or spend weeks researching before their first team conversation. CollectiveStart generates a concrete starting document in under a minute — something the team can mark up, argue about, and bring to a co-op attorney already annotated.

## Tunable AI prompt

The founding starter pack is generated by a single Gemini prompt stored as an `AiTemplate` in the database. To tune the output:

1. Sign in as `demo@example.com` / `password123`
2. Go to Admin → AI Templates → `collectivestart_starter_pack_v1`
3. Edit the system prompt, user prompt, temperature, or token cap
4. Use the built-in test panel to preview output before saving

The prompt is seeded from `db/seed_prompts/collectivestart_starter_pack_v1_system.txt` and `_user.txt`. Edit those files to change the default for new installs.

## Quick Start

1. Clone this repo
2. Run `bin/setup`
3. Add your Gemini API key to `.env`
4. `bin/rails server`
5. Visit http://localhost:3000 and sign in with `demo@example.com` / `password123`

## AI Safety Posture

**What this boilerplate enforces:**
- Per-user daily call cap (default: 50/day, set via `AI_CALLS_PER_USER_PER_DAY`)
- Pre-flight gatekeeper: input length limit, prompt injection patterns, profanity filter
- Hard output token cap per template
- Configurable request timeout (default: 15s)
- Full request log with status, tokens, duration, and cost estimate
- Fail-soft UI: errors render an inline alert, never crash the page
- AI disclaimer in the footer on every page

**Deliberately omitted (with rationale):**
- No PII scrubbing — demo apps have no production user data
- No content moderation API — Gemini's built-in safety filters are sufficient
- No automatic retries — avoids stacking costs on transient failures
- No RAG or vector DB — single-shot prompts only
- No streaming — synchronous calls keep the code simple

See `app/services/ai_gatekeeper.rb` and `app/services/ai_budget_checker.rb` to extend.

## Customization

To build a new demo app on top of this boilerplate:

1. Update `APP_NAME`, `APP_TAGLINE`, `APP_DESCRIPTION` in `.env`
2. Set `--accent` color in `app/assets/stylesheets/application.css`
3. Replace `app/views/home/index.html.erb` with your landing page
4. Add your domain models, controllers, and views
5. Add your `AiTemplate` seeds in `db/seeds.rb`
6. Call `GeminiService.generate(template: "...", variables: {...})` from your controllers

Do not modify the auth system, admin panel, services, or layout for individual demo apps.
